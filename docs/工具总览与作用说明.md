# Hermes-Agent 工具总览与作用说明

## 1. 文档目的

这份文档用于回答两个问题：

1. 这个 Agent 当前注册了哪些工具
2. 每个工具大致负责什么工作

这里说的“工具”，是指会注册进 Hermes 工具注册表、并可能被模型通过 function calling 调用的能力。

说明范围基于当前源码中的注册信息，核心参考位置包括：

- `tools/*.py`
- `tools/registry.py`
- `toolsets.py`
- `model_tools.py`

---

## 2. 先理解：工具、工具集、场景工具集的关系

Hermes 里有三层概念：

### 2.1 单个工具

例如：

- `read_file`
- `terminal`
- `delegate_task`

这是模型真正会调用的最小执行单元。

### 2.2 工具集（toolset）

例如：

- `file`
- `web`
- `terminal`
- `delegation`

工具集是把一组相关工具按主题打包。

### 2.3 场景工具集

例如：

- `debugging`
- `safe`
- `hermes-acp`

这一层不是新增工具，而是把多个基础工具集再组合起来，方便在特定场景快速启用一整套能力。

## 3. Agent 当前注册的核心工具

下面按工具集分类整理。

## 3.1 Web 工具

所属工具集：

- `web`

工具：

- `web_search`
  - 作用：联网搜索信息
  - 典型用途：查资料、找网页、找错误解决方案、收集外部信息

- `web_extract`
  - 作用：提取网页正文或结构化内容
  - 典型用途：抓取文章内容、提取目标网页信息、做网页摘要前的原始读取

适合场景：

- 技术调研
- 查最新信息
- 搜错误原因
- 获取在线文档内容

---

## 3.2 文件工具

所属工具集：

- `file`

工具：

- `read_file`
  - 作用：读取文件内容
  - 典型用途：读源码、读配置、读日志、读文档

- `write_file`
  - 作用：写入或覆盖文件
  - 典型用途：生成文档、创建代码文件、输出配置文件

- `patch`
  - 作用：对文件做补丁式修改
  - 典型用途：局部改源码、修改注释、批量替换小片段

- `search_files`
  - 作用：搜索文件名或文件内容
  - 典型用途：找入口、找方法定义、找配置项、找调用链

适合场景：

- 源码阅读
- 改代码
- 生成功能文档
- 定位某段逻辑在什么文件

---

## 3.3 终端与进程工具

所属工具集：

- `terminal`

工具：

- `terminal`
  - 作用：执行终端命令
  - 典型用途：运行测试、执行脚本、查看 git 状态、安装依赖、构建项目

- `process`
  - 作用：管理和查看后台进程
  - 典型用途：查看某个后台任务是否还在运行、检查进程状态、处理中断后的残留进程

适合场景：

- 运行项目
- 调试命令行问题
- 执行构建/测试
- 管理后台执行任务

---

## 3.4 浏览器自动化工具

所属工具集：

- `browser`

工具：

- `browser_navigate`
  - 作用：打开网页并导航到目标地址

- `browser_snapshot`
  - 作用：获取当前页面结构快照

- `browser_click`
  - 作用：点击页面元素

- `browser_type`
  - 作用：在输入框中输入内容

- `browser_scroll`
  - 作用：滚动页面

- `browser_back`
  - 作用：浏览器回退

- `browser_press`
  - 作用：模拟键盘按键

- `browser_get_images`
  - 作用：提取页面中的图片信息

- `browser_vision`
  - 作用：结合视觉方式理解页面状态

- `browser_console`
  - 作用：读取页面控制台信息

适合场景：

- 自动化操作网页
- 测试页面交互
- 模拟用户点击和输入
- 观察前端页面行为

补充说明：

- `browser` 工具集还会把 `web_search` 一起带上，方便先找 URL，再自动操作网页

- `skill_view`

- `skill_manage`
  - 作用：管理 skill
  - 典型用途：更新 skill、维护 skill、调整技能内容

适合场景：

- skill 学习
- skill 调优
- skill 管理

## 3.7 Todo 与任务规划工具

所属工具集：

- `todo`

工具：

- `todo`
  - 作用：维护任务清单
  - 典型用途：拆步骤、追踪待办、更新完成状态、多步任务管理

适合场景：

---

## 3.8 记忆工具

所属工具集：

- `memory`

工具：

- `memory`
  - 作用：写入或修改持久记忆
  - 典型用途：保存用户偏好、记录长期约束、维护 MEMORY.md 相关内容

适合场景：

- 长期偏好保存
- 跨会话记忆
- 个人配置类信息保存

补充说明：

- Hermes 里“记忆系统”不只等于 `memory` 这个工具
- `memory` 工具更像“主动写记忆”的入口
- 真正的记忆协调还会结合 memory provider、memory manager、后台复盘等机制

适合场景：

## 3.10 澄清提问工具

所属工具集：

- `clarify`

工具：

- `clarify`
  - 作用：向用户发出澄清问题
  - 典型用途：需求不明确时追问、让用户二选一、补齐必要参数

适合场景：

- 任务条件不充分
- 路径选择会带来不同后果
- 需要用户确认关键决策

---

## 3.11 代码执行工具

所属工具集：

- `code_execution`

工具：

- `execute_code`
  - 作用：执行 Python 代码，并可在代码里程序化地调用工具
  - 典型用途：批量处理数据、自动化脚本、减少多轮 LLM 往返

适合场景：

- 机械性批处理
- 大量结构化转换
- 复杂但可脚本化的工作

补充说明：

- 这和 `terminal` 不完全一样
- `terminal` 更偏 shell 命令执行
- `execute_code` 更偏“在受控代码环境里程序化工作”

---

## 3.12 子代理委托工具

所属工具集：

- `delegation`

工具：

- `delegate_task`
  - 作用：生成一个或多个子代理来处理子任务
  - 典型用途：复杂调研、代码审查、并行研究、避免主上下文被中间过程塞爆

适合场景：

- 需要并行做多个子任务
- 中间过程很多，不适合都放进主上下文
- 某个问题需要一个“独立小代理”专门深挖

补充说明：

- 子代理有独立上下文
- 子代理有独立终端会话
- 子代理只把最终摘要回流给父代理
- 子代理默认不能继续递归委托，也不能直接澄清或写共享记忆

如果你正在研究 subagent，这个工具是最核心的入口。

## 4. Agent 默认核心工具有哪些

在 `toolsets.py` 里，有一个 `_HERMES_CORE_TOOLS`，它相当于 Hermes 默认核心能力集合。

这里面包括：

- Web：`web_search`、`web_extract`
- Terminal：`terminal`、`process`
- File：`read_file`、`write_file`、`patch`、`search_files`
- Vision / Image：`vision_analyze`、`image_generate`
- Skills：`skills_list`、`skill_view`、`skill_manage`
- Browser：整套 `browser_*`
- TTS：`text_to_speech`
- Planning / Memory：`todo`、`memory`
- Session：`session_search`
- Clarify：`clarify`
- Code / Delegation：`execute_code`、`delegate_task`
- 定时任务：`cronjob`
- 消息发送：`send_message`
- Home Assistant：`ha_*`

也就是说，从架构上看，Hermes 默认是一个“广谱型任务代理”，而不是只会读写文件的代码助手。

## 6. 如果从源码角度看，哪些工具最重要

如果你是为了学这个 Agent 的核心设计，而不是学所有边缘能力，优先看下面几组。

### 第一优先级：最能体现 Agent 本质的工具

- `terminal`
- `read_file`
- `write_file`
- `patch`
- `search_files`
- `delegate_task`
- `todo`
- `memory`

这些工具直接决定了：

- 它能不能真正执行任务
- 它能不能修改环境
- 它能不能拆任务
- 它能不能长期记忆

### 第二优先级：体现平台扩展能力的工具

- `send_message`
- `cronjob`
- `ha_*`
- `text_to_speech`

这些说明 Hermes 不只是 CLI Agent，还可以扩展成平台化、多模态、自动化系统。

### 第三优先级：体现增强能力的工具

- `browser_*`
- `vision_analyze`
- `execute_code`
- `mixture_of_agents`

这些说明 Hermes 不仅能“调用工具”，还在尝试更高级的自动化与推理模式。

## 7. 学习源码时推荐怎么读这些工具

建议顺序：

1. 先看 `toolsets.py`
   - 理解工具是怎么按场景打包的

2. 再看 `model_tools.py`
   - 理解工具 schema 怎么汇总给模型

3. 再看 `tools/registry.py`
   - 理解工具怎么注册、怎么分发

4. 再按优先级读工具实现
   - `terminal_tool.py`
   - `file_tools.py`
   - `delegate_tool.py`
   - `memory_tool.py`
   - `todo_tool.py`

---

## 8. 最后一句总结

Hermes-Agent 的工具体系不是“给模型挂几个函数”这么简单，而是一整套按场景分层的执行能力系统。

如果只看本质，它最核心的能力可以概括为四类：

- 观察：`read_file`、`search_files`、`web_search`、`vision_analyze`
- 执行：`terminal`、`write_file`、`patch`、`execute_code`
- 协调：`todo`、`delegate_task`
- 持久化与扩展：`memory`、`session_search`、`send_message`、`cronjob`

把这四类看懂了，Hermes 的工具系统你就抓住主干了。
